---
title: 'Get Employee'
api: 'GET /employees/:id'
---

Retrieve detailed information about a specific employee by their ID.

## Authentication

Requires permission: `employee:view`

## Path Parameters

<ParamField path="id" type="number" required>
  Employee ID to retrieve.
</ParamField>

## Response

Returns a complete employee object with all fields and related entities.

```http
GET /employees/supervisors
```

### Get Audit Trail

```http
GET /employees/:id/audit-trail
```

Retrieve the audit trail for an employee, showing all changes made to their record.

**Authentication**: Requires permission `config:employees:audit`

**Path Parameters**:
- `id` (number, required) - Employee ID

**Query Parameters**:
- `limit` (number, optional) - Maximum number of audit entries to return. Defaults to 100.

**Response**: Array of audit trail entries showing what changed, when, and by whom.

**Example Request**:
```bash
curl -X GET "https://api.newkipital.com/employees/1/audit-trail?limit=50" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

## Notes

- Sensitive fields are decrypted before being returned in the response
- Related entities (department, position, supervisor, pay period) are populated with their full details
- The user must have access to the employee's company to retrieve their information
- Audit trail access requires a special permission (`config:employees:audit`) typically reserved for administrators and HR managers
- The audit trail tracks all field-level changes including who made the change and when
