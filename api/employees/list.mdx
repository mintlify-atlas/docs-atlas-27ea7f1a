---
title: 'List Employees'
api: 'GET /employees'
---

Retrieve a paginated list of employees with advanced filtering and sorting capabilities.

## Authentication

Requires permission: `employee:view`

## Query Parameters

<ParamField query="idEmpresa" type="number">
  Filter employees by company ID. Returns only employees belonging to the specified company.
</ParamField>

<ParamField query="idEmpresas" type="string">
  Filter employees by multiple company IDs. Provide comma-separated company IDs (e.g., "1,2,3").
</ParamField>

<ParamField query="includeInactive" type="boolean" default={false}>
  Include inactive employees in the results. By default, only active employees are returned.
</ParamField>

<ParamField query="page" type="number" default={1}>
  Page number for pagination.
</ParamField>

<ParamField query="pageSize" type="number" default={20}>
  Number of employees per page.
</ParamField>

<ParamField query="search" type="string">
  Search employees by name, email, or employee code. Performs a partial match across these fields.
</ParamField>

<ParamField query="idDepartamento" type="number">
  Filter employees by department ID.
</ParamField>

<ParamField query="idPuesto" type="number">
  Filter employees by position/job title ID.
</ParamField>

<ParamField query="estado" type="number">
  Filter by employee status. Use `1` for active employees, `0` for inactive.
</ParamField>

<ParamField query="sort" type="string">
  Field to sort by. Supported values: `nombre`, `apellido1`, `codigo`, `fechaIngreso`, `email`.
</ParamField>

<ParamField query="order" type="string" default="ASC">
  Sort order. Options: `ASC` (ascending) or `DESC` (descending).
</ParamField>

## Example Request

```bash
curl -X GET "https://api.newkipital.com/employees?page=1&pageSize=20&includeInactive=false&sort=apellido1&order=ASC" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

## Notes

- Sensitive fields (cedula, nombre, apellido1, apellido2, email, telefono, direccion, salarioBase, numeroCcss, cuentaBanco, vacacionesAcumuladas, cesantiaAcumulada) are encrypted at rest
- The `datosEncriptados` flag indicates whether the employee's data has been encrypted
- Use the `idEmpresas` parameter for multi-company filtering when users have access to multiple companies
- Supervisor relationships are hierarchical - an employee can have one supervisor
