## Get Catalog Mode

<api method="GET" endpoint="/permissions/mode" auth="config:permissions">
  Returns the current permission catalog mode configuration.

  <ResponseField name="mode" type="string">
    The current catalog mode (e.g., "strict", "permissive")
  </ResponseField>
</api>

## Create Permission

<api method="POST" endpoint="/permissions" auth="config:permissions">
  Creates a new atomic permission in the system. Permissions represent specific actions that can be performed.

  <ParamField body="codigo" type="string" required>
    Unique permission code following the format `module:action[:subaction]` in lowercase. Must be 3-100 characters. Examples: `employees:list`, `payroll:approve`, `company:edit:all`.
  </ParamField>

  <ParamField body="nombre" type="string" required>
    Display name for the permission. Maximum 150 characters.
  </ParamField>

  <ParamField body="descripcion" type="string">
    Optional description explaining what this permission allows. Maximum 300 characters.
  </ParamField>

  <ParamField body="modulo" type="string" required>
    Module this permission belongs to (e.g., "employees", "payroll", "company"). Maximum 50 characters, lowercase.
  </ParamField>

  <ResponseField name="id" type="number">
    Auto-generated unique identifier for the permission
  </ResponseField>

  <ResponseField name="codigo" type="string">
    The permission code as provided
  </ResponseField>

  <ResponseField name="nombre" type="string">
    The permission display name
  </ResponseField>

  <ResponseField name="descripcion" type="string | null">
    Permission description if provided
  </ResponseField>

  <ResponseField name="modulo" type="string">
    Module identifier
  </ResponseField>

  <ResponseField name="estado" type="number">
    Permission status: 1 = active, 0 = inactive
  </ResponseField>

  <ResponseField name="fechaCreacion" type="string">
    ISO timestamp of when the permission was created
  </ResponseField>

  <ResponseField name="fechaModificacion" type="string | null">
    ISO timestamp of last modification
  </ResponseField>

  <ResponseField name="creadoPor" type="number | null">
    User ID who created the permission
  </ResponseField>

  <ResponseField name="modificadoPor" type="number | null">
    User ID who last modified the permission
  </ResponseField>
</api>

</api>

## Inactivate Permission

<api method="PATCH" endpoint="/permissions/:id/inactivate" auth="config:permissions">
  Marks a permission as inactive without deleting it. Inactive permissions cannot be assigned to roles.

  <ParamField path="id" type="number" required>
    The ID of the permission to inactivate
  </ParamField>

  <ResponseField name="id" type="number">
    Permission ID
  </ResponseField>

  <ResponseField name="estado" type="number">
    Permission status (will be 0 for inactive)
  </ResponseField>

  <ResponseField name="fechaModificacion" type="string">

  </ResponseField>
</api>

## Permission Code Format

Permissions follow a hierarchical naming convention:

- **Format**: `module:action[:subaction]`
- **Rules**:
  - Must be lowercase
  - Use hyphens or underscores for word separation
  - At least two segments (module:action)
  - Optional sub-actions for fine-grained control

**Examples**:

```
employees:list
employees:create
employees:edit
employees:delete
payroll:view
payroll:approve
payroll:approve:all
company:edit:settings
company:edit:billing
```
