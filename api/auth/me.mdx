---
title: "Get Current User"
api: "GET /api/auth/me"
description: "Retrieve the current authenticated user's session information"
---

Retrieves complete session information for the currently authenticated user, including user details, companies, permissions, and roles. Optionally scoped to a specific company and application.

## Endpoint

```
GET /api/auth/me
```

## Authentication

This endpoint requires a valid JWT access token. The token should be present in the `kpital_token` httpOnly cookie.

**Guard:** JwtAuthGuard

## Query Parameters

<ParamField query="companyId" type="string">
  Optional company ID to scope the session context. When provided, permissions and roles are filtered for this specific company.
</ParamField>

<ParamField query="appCode" type="string">
  Optional application code to scope the session context (e.g., "kpital", "analytics"). When provided, permissions and roles are filtered for this specific application.
</ParamField>

<ParamField query="refreshAuthz" type="string">
  Optional flag to bypass permission cache and fetch fresh authorization data. Accepts: "1", "true", "yes", "y", "on"
  
  Default: false (uses cached permissions)
</ParamField>

  <ResponseField name="companyIds" type="string[]" required>

  Array of permission codes the user has access to. Scoped to the specified company and app if provided.

<ResponseField name="roles" type="string[]" required>
  Array of role names assigned to the user. Scoped to the specified company and app if provided.
</ResponseField>

## Request Examples


```bash
GET /api/auth/me?companyId=1&appCode=kpital&refreshAuthz=true
```

## Error Responses

### 401 Unauthorized - No Token

Returned when no valid authentication token is present.

```json
{
  "statusCode": 401,
  "message": "Unauthorized",
  "error": "Unauthorized"
}
```

### 401 Unauthorized - User Not Found

Returned when the authenticated user no longer exists in the database.

```json
{
  "statusCode": 401,
  "message": "Usuario no encontrado",
  "error": "Unauthorized"
}
```

### 400 Bad Request - Invalid Company ID

Returned when the companyId parameter is not a valid integer.

```json
{
  "statusCode": 400,
  "message": "Validation failed (numeric string is expected)",
  "error": "Bad Request"
}
```

## Permission Resolution

Permissions are resolved using a complex authorization system:

1. **User-Company-Role assignments** - Roles assigned to the user for specific companies
2. **User-Role-Global assignments** - Roles assigned globally across all companies
3. **User-Role-Exclusions** - Specific role exclusions for companies
4. **User-Permission-Overrides** - Direct permission grants/denies at the user level
5. **User-Permission-Global-Deny** - Global permission denials across all companies

The final permission set is calculated by combining all these rules with proper precedence.

## Caching

Permissions are cached for performance:

- **Cache Key:** Based on user ID, company ID, and app code
- **Cache Bypass:** Use `refreshAuthz=true` to force fresh data
- **Cache Invalidation:** Automatically invalidated when permissions change

## Notes

- The access token is automatically validated by the JwtAuthGuard
- Permissions are calculated dynamically based on the current authorization rules
- The `companyId` and `appCode` parameters allow filtering permissions by context
- Use `refreshAuthz=true` when you need the most up-to-date permission data
- This endpoint is commonly used to verify authentication status and load user context
